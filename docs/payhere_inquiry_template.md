## 서비스 소개

저희는 **비건 베이커리 전용 모바일 앱**을 개발 중입니다. 이 앱은 고객들이 매장의 실시간 재고를 확인하고 사전 예약/주문을 할 수 있는 서비스입니다.

**현재 문제점**:
- 네이버 예약 등 기존 플랫폼은 매장의 오프라인 판매와 재고가 연동되지 않음
- 고객이 품절된 빵을 예약하는 문제 발생

**해결 방안**:
- 페이히어 POS와 저희 앱을 API로 연동하여 **실시간 재고 동기화** 구현
- 오프라인에서 빵이 팔리면 → 온라인 앱에 즉시 반영

## 기술적 요구사항

### 1. Webhook (권장)
페이히어에서 **결제 완료 시 Webhook**을 저희 서버로 전송:

```
POST https://our-server.com/api/pos/webhook
Content-Type: application/json

{
  "event": "payment.success",
  "store_id": "xxx",
  "items": [
    {"product_id": "bread_001", "name": "소금빵", "quantity": 2, "price": 3500}
  ],
  "timestamp": "2024-12-21T10:30:00+09:00"
}
```

### 2. API 조회 (대안)
판매 데이터를 주기적으로 조회하는 API 제공:

```
GET /api/sales?store_id=xxx&since=2024-12-21T00:00:00
```

## 확인이 필요한 사항

1. **개별 매장 API 연동 가능 여부**
   - 프랜차이즈가 아닌 단일 매장도 API 연동이 가능한가요?

2. **제공되는 연동 방식**
   - Webhook 방식 지원 여부
   - REST API 제공 여부
   - 개발자 문서(Documentation) 제공 여부

3. **비용**
   - API 연동 시 추가 비용이 있나요?
   - 월정액 또는 트랜잭션 기반 요금제가 있나요?

4. **데이터 범위**
   - 판매 내역 (상품명, 수량, 시간)
   - 상품 마스터 데이터 (상품 목록, 가격)
   - 실시간 이벤트 vs 배치 데이터

5. **개발 지원**
   - 샌드박스/테스트 환경 제공 여부
   - 기술 지원 채널
