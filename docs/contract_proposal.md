## 1. 프로젝트 개요

### 1.1 서비스 설명

비건베이커리 운영을 위한 모바일 앱 서비스를 개발합니다.

| 구분 | 내용 |
|---|---|
| **고객용 앱** | 메뉴 확인, 실시간 재고, 주문/결제, 픽업, 푸시 알림 |
| **사장님용 앱** | 재고 관리, 주문 관리, 출고 알림 발송, 매출 현황 |
| **핵심 기능** | 빵드롭 (실시간 재고 알림), 사전 주문, QR 픽업 |

### 1.2 기술 스택

| 영역 | 기술 | 비고 |
|---|---|---|
| 모바일 앱 | React Native (Expo) | iOS/Android 동시 개발 |
| 백엔드 | FastAPI (Python) | 비즈니스 로직 처리 |
| 데이터베이스 | Supabase (PostgreSQL) | 실시간 동기화 포함 |
| 결제 | PortOne | 카카오페이, 네이버페이 |
| 푸시 알림 | Firebase FCM | 출고/주문 알림 |

---

## 2. 개발 범위 및 산출물

### 2.1 MVP 단계 (Phase 1)

#### 고객용 앱
- [ ] 홈 화면 (오늘의 빵, 다음 출고 시간)
- [ ] 메뉴 목록 및 상세 정보
- [ ] 실시간 재고 표시 (빵드롭)
- [ ] 장바구니 및 주문
- [ ] 결제 (카카오페이/네이버페이)
- [ ] 주문 내역 및 QR 픽업권
- [ ] 푸시 알림 수신 (출고/주문 상태)
- [ ] 마이페이지 (알림 설정)

#### 사장님용 앱
- [ ] 대시보드 (오늘 매출, 주문 현황)
- [ ] 재고 관리 (수량 조절, 품절 처리)
- [ ] 출고 등록 및 알림 발송
- [ ] 주문 관리 (확인/준비중/완료)
- [ ] QR 스캔 픽업 확인

#### 백엔드/인프라
- [ ] FastAPI 서버 개발
- [ ] Supabase 데이터베이스 설계 및 구축
- [ ] 실시간 구독 설정 (Supabase Realtime)
- [ ] 결제 연동 (PortOne)
- [ ] 푸시 알림 연동 (Firebase FCM)
- [ ] 서버 배포 (Vercel/Railway)

### 2.2 산출물

| 산출물 | 설명 |
|---|---|
| iOS 앱 2개 | 고객용, 사장님용 (App Store 배포) |
| Android 앱 2개 | 고객용, 사장님용 (Play Store 배포) |
| 백엔드 서버 | FastAPI 기반 REST API |
| 관리자 대시보드 | 웹 기반 (옵션) |
| 기술 문서 | API 문서, 데이터베이스 스키마 |
| 운영 가이드 | 앱 사용 매뉴얼 |

## 3. 개발 일정

### 3.1 예상 기간: **8~10주**

```
주차    작업 내용                          산출물
────────────────────────────────────────────────────
1주     프로젝트 셋업, DB 설계             개발환경, 스키마
2주     인증 구현                          카카오/애플 로그인
3주     메뉴/재고 기능                     메뉴 목록, 실시간 재고
4주     장바구니, 결제 연동                PortOne 연동
5주     주문 플로우 완성                   주문→픽업 전체 흐름
6주     푸시알림 구현                      출고/주문 알림
7주     사장님 앱 기능                     재고/주문 관리
8주     테스트 및 버그 수정                QA 완료
9-10주  앱스토어 심사 및 배포              정식 출시
────────────────────────────────────────────────────
```

### 3.2 마일스톤

| 단계 | 완료 시점 | 지급 조건 |
|---|---|---|
| 기획/디자인 확정 | 계약 후 1주 | 착수금 지급 |
| 1차 프로토타입 | 계약 후 4주 | 중도금 1차 지급 |
| 기능 개발 완료 | 계약 후 7주 | 중도금 2차 지급 |
| 정식 출시 | 계약 후 10주 | 잔금 지급 |

## 8. 특약 사항

### 8.1 권리 귀속

- 개발 완료 후 **소스코드 소유권**은 클라이언트에게 귀속됩니다.
- 개발자는 **포트폴리오 활용 권리**를 갖습니다. (앱 스크린샷, 기술 설명)

### 8.2 기밀 유지

- 프로젝트 진행 중 취득한 영업 비밀, 고객 정보 등은 외부에 공개하지 않습니다.

### 8.3 일정 변경

- 기획 변경, 추가 요구사항 발생 시 일정 및 비용이 조정될 수 있습니다.
- 클라이언트의 피드백 지연 시 일정이 연장될 수 있습니다.

### 8.4 앱스토어 심사

- Apple/Google 심사 리젝 시 수정하여 재제출하며, 심사 기간은 개발 일정에 포함되지 않습니다.
